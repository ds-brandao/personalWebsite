## Build and Development Commands

All commands should be run from the `next-app/` directory:

```bash
# Install dependencies
npm install

# Development server (http://localhost:3000)
npm run dev

# Production build
npm run build

# Type check and lint
npm run lint
npx tsc --noEmit

# Pre-generate AI project analyses (requires OPENAI_API_KEY in .env.local)
npm run seed
```

### Docker Development

```bash
# From repository root
docker compose up dev          # Dev server at http://localhost:3000
docker build -t personal-website .  # Production build
docker run -p 80:80 personal-website
```

## Architecture Overview

This is a Next.js 16 personal portfolio website using TypeScript, Tailwind CSS, and server-side rendering.

### Data Flow

- **Configuration**: Personal info, social links, and tag definitions loaded from `public/config/config.json` via `getConfig()` (server-side file read)
- **Articles**: Blog metadata loaded from `public/config/articles.json` via `getArticles()`; actual content is Markdown files in `public/blog-posts/`
- **Projects**: GitHub repos fetched server-side via `getGitHubRepos()` with local `public/config/projects.json` as fallback
- **Project Analyses**: Pre-generated AI analyses read from `public/config/analyses.json` (generated by `npm run seed`)
- **Commits**: Recent commits per repo fetched server-side via `getRepoCommits()`

### Key Components

- **app/page.tsx**: Server component that fetches all data and passes to PageClient
- **PageClient**: Client component orchestrating the full page layout
- **ArticleCard**: Blog post cards with tag filtering
- **Projects**: Horizontal scrollable project cards with detail panel
- **ProjectDetail**: Renders pre-computed AI analyses (package info, code snippets, file structure) and recent commits

### Type Definitions

Core types in `types/index.ts`:
- `Config`: Site configuration with personal, social, and tags sections
- `Article`: Blog post metadata with markdown path and tag array
- `GitHubRepo`: Repository data from GitHub API
- `GitHubCommit`: Commit data (sha, message, author, date)
- `ProjectAnalysis`: Pre-generated AI analysis with tool results
- `ToolResult`: Individual AI tool output (packageInfo, codeSnippet, fileStructure, setupCommand)

### AI Analysis Seed Script

`scripts/seed-analyses.ts` generates `public/config/analyses.json` by:
1. Reading config for GitHub username
2. Fetching repos from GitHub API
3. Calling OpenAI gpt-4o-mini to analyze each repo
4. Writing structured results (package info, code snippets, file trees, setup commands)

Run manually with `npm run seed` when you want fresh analyses.

## Deployment

Self-hosted runner on main branch. GitHub Actions workflow:
1. Builds multi-stage Docker image (node:20-alpine → standalone output)
2. Deploys container on port 3000
3. Tests endpoints
